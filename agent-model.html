05 · THE AI AGENT OPERATING MODEL

Agents do the work. Humans own the risk.

The 2026 shift isn't “AI writes code.” It's that operations itself is becoming agentic — triage, remediation, cost hygiene, and compliance checks run as supervised agents. The IT manager's new deliverable is the governance layer around them.

🎚️

Graduated autonomy

Agents earn permissions like employees do. Read-only diagnostics first; reversible actions with approval next; auto-remediation only for proven, low-blast-radius patterns.

🧾

Every action is a change record

Agent actions flow through the same PR/approval pipeline as human changes. An agent that can't produce an audit trail doesn't get production access. Ever.

🔌

Tools over prompts (MCP)

Agents reach systems through governed tool interfaces — Model Context Protocol servers with scoped credentials — not through pasted API keys and hope.

🧯

Kill switches & blast-radius budgets

Every automation has a disable flag, a rate limit, and a maximum scope. The failure mode of an agent must always be “stopped,” never “creative.”

📏

Agents get KPIs too

Precision of triage, false-page rate, savings realized, evidence completeness. If an agent's numbers slip, it gets retrained or retired — same as any process.

🧑‍⚖️

Human-in-the-loop by design

Approval gates live where the risk lives: production changes, spend actions, customer-facing comms. Humans decide; agents execute and document.
